发表于 2017-10-23

React Native 实现步骤

​ 在整个应用设计中，始终按照自下而上的原则进行。

基础组件结构设计

​ 首先在 UI 界面上找出所有需要使用的基础组件。基础组件需要遵循“责任唯一原则”。意思是一个基础组件只应当负责一个工作，如果一个组件负责多个工作，就应当将这个组件拆分成多个子组件。

​ 对基础组件进行分层。分层就是把显示在同一行区域或列区域中的多个组件放在同一个 View 中。

​ 从 UI 界面的底部（最基础的 React Native 组件）开始构建，每分出一层组件构成了一个上层 React Native 组件，直到向上分层到一个界面的根 View 组件。

React Native 组件分层

  1. 分析界面之间的联系
  2. 确定数据的显示者与拥有者
  3. 确定状态机变量
  4. 确定各事件的接受者与处理者
  5. 非界面触发事件的接收者与处理者（后退键监听、混合开发消息监听等）

实现各组件业务逻辑

​ 业务逻辑包括但不限于：

  1. 直接渲染状态机变量和属性

  2. 将状态机变量以属性的方式交给下层组件

  3. 实现状态机变量的其他业务控制逻辑

  4. 无直接关系的 React Native 组件间的传递：

    ​ 通过无直接关系的公共父组件来间接传递消息。对于这种间接传递消息，消息发送者不需要指定消息接受者，由公共父组件按业务逻辑来判定消息接受者是谁。
