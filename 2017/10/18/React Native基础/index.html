状态机

  • 通过绑定状态机变量，当组件的状态机发生改变时（即调用 this.setState() ），所有使用到状态机变量的组件将重新渲染。——观察者模式

  • 设计思路：尽可能减少有状态的 React Native 组件的数目，通过有状态的 React Native 组件的状态机变量的值通过 props 传给无状态的 React Native 组件。这样，有状态的 React Native 组件封装了 UI 的交互逻辑，而无状态的 React Native 组件负责渲染 UI 界面。

  • 使用 onChangeText = { this.updateNum } 这种方式进行回调时，updateNum 函数中的 this 将不是指向组件的实例，因此，无法解析正确的 this，需要对 update、Num 进行绑定。

混合开发

  1. Android 平台混合开发

    ​ Android 侧的接口类需要继承 ReactContextBaseJavaModule 类，使用 “@ReactMethod” 注解时函数可以被 React Native 侧调用，并且这些函数不能有返回值。

    ​ 建立一个 React 包管理类，这个类继承自 ReactPackage，并且实现 creatNativeModules 方法，将原生代码模块进行注册。

    ​ 最后，把 React 包管理类在ReactApplication 中实例化。

  2. Android 与 React Native 的消息传递（Promise 机制）

    ​ React Native 通过调用 Android 原生代码，传递信息；React Native 先监听原生信息，注册回调函数（React Native 调用原生代码是异步执行的），Android 原生代码完成处理后，向 React Native 侧发送信息，回调函数需要先注册，Android 侧无法主动回调，并且回调函数只能执行一次（需要保存 Promise 才能多次调用）。
